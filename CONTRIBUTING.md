# Contributing to node-rtc-connection

Thanks for your interest in contributing! This project is a from-scratch,
pure-Node.js WebRTC data-channel stack written in strict TypeScript. Bug
reports, fixes, tests, and documentation improvements are all welcome.

## Getting started

```bash
git clone https://github.com/nmhung1210/node-rtc-connection.git
cd node-rtc-connection
npm install
npm run typecheck   # strict tsc over src + tests
npm test            # full suite
```

- **Node.js 18+** is required.
- Source lives in `src/**/*.ts`; tests in `test/**/*.ts` and run through
  [`tsx`](https://github.com/privatenumber/tsx) (no precompile step).
- `dist/` is generated by `npm run build` (rollup) — never edit it by hand.

## Development workflow

1. Create a branch off `main`.
2. Make your change. Keep it focused; match the surrounding code style.
3. Add or update tests for any behavior change.
4. Run `npm run typecheck` and `npm test` until green.
5. Open a pull request describing **what** changed and **why**.

### Useful scripts

| Command | What it does |
| --- | --- |
| `npm run build` | Bundle `src/` → `dist/` (minified CJS + ESM) and emit `.d.ts` |
| `npm run typecheck` | Strict `tsc --noEmit` over source and tests |
| `npm test` | Full suite; auto-starts a coturn container for the TURN test |
| `npm run test:unit` | `SKIP_INTEGRATION=1` — no Docker / browser / external servers |
| `npm run test:coverage` | Full suite under c8 |
| `npm run test:ci` | Reproduce CI locally (ensures Playwright Chromium + coturn) |

## Testing & verification

This project is verified against **external references**, not just self-loopback:

- **DTLS** handshakes against `openssl s_server` / `s_client`.
- **Browser interop** drives real Chromium via Playwright.
- **TURN relay** runs against a real `coturn` server (started automatically by
  the test runner via Docker, or provided in CI).

Integration suites skip gracefully when their dependency (Docker, openssl,
Chromium) is unavailable or when `SKIP_INTEGRATION=1`. At minimum, please run
`npm run typecheck` and `npm run test:unit` before submitting; run the full
`npm test` if your change touches the transport stack.

New behavior should keep coverage from regressing — see thresholds in
`.c8rc.json`.

## Coding conventions

- **Strict TypeScript**, ES module syntax (`import`/`export`) in source.
- Class internals use ECMAScript hard-private `#` fields/methods (not the TS
  `private` keyword or `_` prefix).
- Protocol layers extend `EventEmitter` and communicate via events.
- Don't add runtime dependencies: the stack is intentionally pure Node
  (`crypto`, `dgram`, `net`, …) with no native or third-party deps.
- Preserve the existing acronym casing conventions (e.g. `DtlsConnection`,
  `SctpAssociation`, but `OID`/`TSN`/`PPID` as data-format acronyms).

## Commit messages & PRs

- Use short, imperative commit subjects ("Add SCTP heartbeat handling").
- Reference related issues where applicable.
- Keep PRs scoped to one logical change.

## Reporting security issues

Please do **not** open public issues for security vulnerabilities. See
[SECURITY.md](./SECURITY.md).
